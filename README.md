

# 🚀 Точечная маршрутизация для OpenWrt (WireGuard / AmneziaWG)

Этот скрипт полностью автоматизирует настройку точечного обхода блокировок (Split Tunneling) на роутерах с OpenWrt. Он позволяет пускать через VPN только выбранные заблокированные сайты (например, Instagram, YouTube, X), оставляя весь остальной домашний трафик напрямую через вашего провайдера. 

Скрипт написан с упором на **максимальную отказоустойчивость** и прощает любые обрывы связи во время установки.

---

### ✨ Главные фичи (Почему этот скрипт "Неубиваемый")

* **Поддержка AmneziaWG:** Умеет автоматически ставить и настраивать форк WireGuard с защитой от DPI-блокировок (через интеграцию с установщиком *Slava-Shchipunov*).
* **Абсолютная защита от сбоев (Броня):** Любые скачивания пакетов и списков репозиториев обернуты в циклы с проверками. Если ваш провайдер блокирует пакетный менеджер OpenWrt — скрипт будет долбить стену, пока не скачает всё необходимое.
* **Механизм самоочистки:** Скрипт можно запускать хоть 10 раз подряд без сброса роутера. Перед началом работы он аккуратно удаляет старые службы, зоны фаервола и правила маршрутизации, чтобы не плодить дубликаты.
* **Умный Hotplug:** Созданная служба маршрутизации сама следит за состоянием сети. Если провайдер оборвет связь, роутер автоматически восстановит все правила перехвата трафика при переподключении.
* **Эстетика системы:** Скрипт не мусорит в системе. Он создает аккуратные логические интерфейсы (`WG` или `AWG_VPN`) и чистые зоны фаервола, динамически определяя скрытые физические имена туннелей в ядре Linux.
* **Агрессивный переход на dnsmasq-full:** Корректно удаляет базовый DNS-сервер и ставит полную версию с поддержкой `nftset` (без конфликтов файлов конфигурации).

---

### ⚙️ Системные требования и ограничения

Скрипт рассчитан на современные домашние роутеры и использует актуальную маршрутизацию `nftables`.

1. **Версия OpenWrt:** Строго стабильные релизы (Releases) версий **22.03, 23.05 или 24.10**. Скрипт **НЕ работает** на ежедневных сборках (Snapshots) из-за несовпадения хэшей ядер для пакетов `kmod`.
2. **Память (Flash):** Не менее **16 МБ** (рекомендуется 32 МБ). Установка `dnsmasq-full` и криптографических библиотек не влезет на старые роутеры с 4/8 МБ памяти.
3. **Оперативная память (RAM):** Не менее **128 МБ** (работа с большими списками доменов в памяти требует ресурсов).
4. **Архитектура Firewall:** Используется `fw4` (nftables). На старых прошивках с `fw3` (iptables) маршруты не поднимутся.
5. *Примечание для кастомных прошивок (FriendlyWrt, ImmortalWrt):* Установка стандартного WireGuard пройдет успешно. Для AmneziaWG может потребоваться предварительная ручная установка пакета `kmod-amneziawg`, скомпилированного под ядро конкретной прошивки.

---

### 🛠 Как это работает (Под капотом)

1. Вы задаете список доменов в конфиге `dnsmasq` (внутри скрипта).
2. Когда ваше устройство (телефон/ТВ) запрашивает IP-адрес заблокированного сайта, `dnsmasq` мгновенно складывает полученный IP в специальную "корзину" `nftset` внутри фаервола.
3. Служба `vpn-routing` помечает весь трафик, идущий к адресам из этой корзины, специальной меткой (`mark 0x1`).
4. Правила `ip rule` перехватывают помеченный трафик и отправляют его в таблицу маршрутизации `100`, где шлюзом по умолчанию является ваш VPN-туннель.

### 🚀 Быстрый старт

Подключитесь к роутеру по SSH и выполните одну команду:

```bash
wget --no-check-certificate -O /tmp/setup.sh https://raw.githubusercontent.com/BlueG1ue/openwrt_point-to-point_routing/main/setup.sh && sh /tmp/setup.sh
```


После завершения работы скрипта:

1. Зайдите в веб-интерфейс LuCI (Сеть -> Интерфейсы).

2. Нажмите Редактировать на созданном интерфейсе (WG или AWG_VPN).

3. Вставьте ваши ключи и IP-адрес сервера.

4. Нажмите Сохранить и применить.
